# Clean Code — быстрый чеклист

## 1) Naming
- Имя должно объяснять намерение.
- Избегай аббревиатур без контекста.
- Используй один термин для одной сущности во всем проекте.

## 2) Functions
- Функции должны быть маленькими и делать одну вещь.
- Аргументов лучше не больше 2-3.
- Избегай скрытых side effects.

## 3) Comments
- Комментарий пишется там, где код не может быть самодокументируемым.
- Удаляй устаревшие комментарии сразу.

## 4) Error Handling
- Выделяй ошибки в отдельные сценарии.
- Не смешивай happy path и обработку ошибок в один большой блок.

## 5) Refactoring Routine
- Найди дублирование.
- Упрости имена.
- Разбей длинные функции.
- Добавь/обнови тест.
